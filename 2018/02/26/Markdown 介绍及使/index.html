Markdown 介绍及使用

开始写起了Blog，不得不学的一个技术就是Markdwon。对于Markdown，其实自己也不是第一次接触。从很久以前就使用过有道云笔记、印象笔记这些记录工具，后来加入了GitHub交友网站后，每一次写Readme也是使用Markdown，虽然只是写几句话，从来没真正寻其内涵。这一次自己开通了GitHub Pages，也就趁机好好学一下Markdwon。所以，自己的第一篇文章也就从Markdown开始吧。

初识Markdown

Markdown是由约翰·格鲁伯（John Gruber）和亚伦·斯沃茨（Aaron Swartz）发明的一种轻量级的「标记语言」。它的语法大方简洁，上手非常简单，目前被越来越多的写作爱好者，撰稿者和工程师广泛使用（https://www.imooc.com/article/12063，来源：慕课网 ）。Markdwon的作者之一John Gruber这样解释这种语言：

我自己的体会就是，Markdwon作为一种输入语法，让一些常见的编辑格式通过规定好的编辑符号表现出来，使文字的排版与编辑更加快捷方便。与日常使用的“所见即所得”的编辑方式不同，Markdwon的编辑不仅仅是文字，还有包含有许多排版格式符号，也是这些符号让文章的排版简介快速化，让手不用离开键盘就能完成整个文档的输入和格式编辑工作。这一点有点类似平时在写论文的时候运用的 LaTex语言，不过LaTex更多针对的是具有复杂公式，需要很多图片排版的科技论文，而Markdown针为更普遍的日志文书类的应用，语法会更加简单明了。

既然提到了作者John Gruber，也不得不提到另一个作者Aaron Swartz。我在查阅Markdown的历史的时候，才了解了了Aaron Swartz的故事。这是一个天才的故事，也是IT届乃至整个世界的一个损失。在自己的博客上我就不提政治之类的东西了，但是我至少知道Aaron Swartz是一个为自己的理想和理念努力奋斗的人。硅谷密探有一篇文章写得很好《16岁的黑客天才，26岁他自杀了，中间是..》（https://baike.baidu.com/tashuo/browse/content?id=b7f93dd6bbf5ecd7aa3eafe8&lemmaId=&fromLemmaModule=pcBottom ），介绍了这位天才的一生，有机会的时候我也会多看看这篇文章，向前辈致敬。这段最后还是要致敬一下两位天才的前辈。

标题作为文章常出现的文字格式，在Markdown中设置的方法很简洁，只需要#加文字标题内容即可。针对于不同级别的标题，Markdown采用加#的方式进行排列，从最大到最小有六级标题。

对于文字内容的强调，Markdown也有很简介的设置。具体效果有加粗、下划线、斜体三种表现形式

换行

换行为四个及以上空格加回车。我个人感觉还是自己回车两次，多空一行会更好看。这一点有点类似于Latex语法习惯。

在Mweb中，如果不想打这么多空格，只要回车就为换行，请勾选：Preferences - Themes - Translate newlines to <br> tags
